# 监测页面共享设备状态设计

## 目标

以温湿度传感器页右上角状态为唯一视觉基准，让温湿度、风速、降雨、振动和视频监控页面使用一致的在线状态排版，同时保留视频页的模型就绪与视频源在线信息。

## 方案选择

采用共享 Vue 组件 `DeviceOnlineStatus`。五个页面向组件传入状态、显示文案和最后通讯时间，组件统一负责状态点、颜色、两行布局与时间格式。

未采用以下方案：

- 在五个页面分别复制 CSS：初始改动少，但现有样式偏差正是重复实现造成的。
- 只抽取全局 CSS 类：能统一外观，但时间格式、离线状态和空时间处理仍会分散在各页面。

## 共享组件

`DeviceOnlineStatus.vue` 接收以下信息：

- `status`：`online` 或 `offline`。
- `statusText`：主状态文案；传感器为“在线/离线”，视频为“视频源在线/视频源离线”。
- `timestamp`：Unix 秒或毫秒时间戳。
- `emptyTimeText`：没有通讯时间时的显示文案，默认“暂无通讯”。

组件视觉严格沿用温湿度页面：右对齐，两行布局；第一行为 10px 发光状态图标和 16px 加粗状态文字，第二行为 11px 弱化文字 `最后通讯: MM/DD HH:mm:ss`。在线使用系统成功色，离线使用系统危险色。状态图标使用项目已安装的 Element Plus 图标，不新增图片资源。

组件内部统一识别秒和毫秒时间戳，并使用固定的 `MM/DD HH:mm:ss` 格式，避免浏览器区域设置产生多余空格或不同日期格式。无时间时仍显示第二行，明确展示“最后通讯: 暂无通讯”，避免页面跳动。

## 传感器页面接入

温湿度、风速、降雨和振动页面均用共享组件替换当前重复的 `header-status` 模板和样式：

- 保留各页现有在线/离线判断。
- 保留各页真实采集时间作为最后通讯时间。
- 删除重复的 `formatCommTime` 和状态 CSS。
- 振动页不再把状态与最后通讯排在同一行，状态文案统一为简短的“在线/离线”；模拟数据或连接失败仍通过离线颜色表达。

## 视频页面整合

视频页不删除模型状态和视频源状态的信息，只重组表现：

1. 模型状态保留在顶部，去掉胶囊边框和背景，改为紧凑的次级状态文字与状态图标，继续显示“目标检测模型就绪/未加载”或“图片分类模型就绪/未加载”。
2. 视频源状态由共享组件承担，主文案为“视频源在线/视频源离线”。
3. 最后通讯时间取后端已有的 `currentCamera.last_frame_time`，代表最近一帧到达时间。
4. 移除“当前时间”卡片和每秒时钟定时器；本机时间不再冒充设备通讯时间。
5. 视频页没有选中摄像头时显示“视频源离线”，第二行显示“最后通讯: 暂无通讯”。

## 响应式行为

- 桌面端状态组保持在标题右侧，模型状态在前、视频源状态在后。
- 窄屏时状态组允许换行，但共享状态组件仍保持自身两行右对齐；传感器页纵向布局时与标题区保持一致间距。
- 不改变页面其他卡片、图表、视频播放和 AI 操作逻辑。

## 测试与验收

- 单元测试覆盖时间戳秒/毫秒转换、固定时间格式、无时间占位以及在线/离线文案。
- 前端完整测试和生产构建通过。
- 五个页面的状态区在相同状态下视觉一致。
- 视频页模型信息仍可见；视频源在线状态和最后一帧时间随 3 秒状态刷新更新；不再显示当前时间卡片。
- 对照用户提供的温湿度状态截图完成桌面端视觉检查，确认对齐、字号、间距、颜色和发光效果。

## 范围边界

- 本次不修改后端接口；视频接口已提供 `last_frame_time`。
- 本次不改变设备在线判定阈值。
- 本次不修改状态区以外的业务页面布局。

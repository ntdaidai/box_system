# 统一登录与七天开发会话设计

## 目标

修正当前“系统大部分页面可直接进入，只有视频监控突然要求登录”的割裂流程。在开发阶段，登录后的 JWT 会话有效期设为 7 天（604800 秒）；只有未登录、令牌失效或用户主动退出时进入登录页。

## 现状与根因

- `/monitor/camera` 单独设置了 `requiresAuth`，其他业务页面没有统一路由保护。
- 页面右上角在没有用户信息时固定显示 `admin`，未登录状态看起来像已登录。
- 用户仓库已有 `logout()`，但页面没有退出入口。
- 后端 JWT 的真实有效期来自 `JWT_EXPIRE_SECONDS`，登录响应却把 `expires_in` 固定为 86400 秒。
- 视频接口继续要求 Bearer token；取消该认证会暴露摄像头、上传和 AI 检测能力，不在本次范围内。

## 方案选择

采用全系统统一前端认证：登录页是唯一公开页面，应用主布局统一声明需要认证。路由守卫根据所有匹配路由记录判断权限；没有本地 token 时跳转登录，携带安全的原目标地址。存在 token 时允许进入，并通过 `/api/auth/info` 恢复真实用户信息；服务端返回 401 时沿用全局拦截器清理会话并返回登录页。

未采用以下方案：

- 继续只保护视频页面：实现简单，但保留当前割裂体验。
- 去掉视频接口认证：体验表面统一，但会公开敏感设备与推理接口。

## 前端设计

1. 根布局路由设置 `requiresAuth: true`，视频子路由不再承担特殊登录语义。
2. 路由判断提取为纯函数，覆盖父路由权限继承、安全重定向和登录后回跳，便于 Node 单元测试。
3. 用户仓库新增一次性会话恢复：有 token 时请求 `/auth/info`，将真实用户写入 `userInfo`；失败由现有 401 拦截器统一处理。
4. 顶栏头像改成下拉菜单，展示真实用户名；点击“退出登录”清除 token、用户信息和本地响应缓存，然后跳转 `/login`。
5. 不再使用 `admin` 作为未认证状态的视觉兜底，避免伪登录状态。

## 后端与配置设计

1. Docker Compose 开发配置将 `JWT_EXPIRE_SECONDS` 设置为 `604800`。
2. 后端配置默认值和 `.env.example` 同步为 `604800`，避免脱离 Compose 启动时回退到 24 小时。
3. 登录响应的 `expires_in` 直接读取 `settings.JWT_EXPIRE_SECONDS`，保证响应元数据与 JWT 的 `exp` 一致。
4. JWT 密钥保持不变，因此后端容器重启不会主动使仍在有效期内的 token 失效。

## 数据流与异常处理

1. 用户访问受保护页面且没有 token：路由守卫转到 `/login?redirect=<原地址>`。
2. 登录成功：保存 token 和返回的用户信息，回到安全的站内目标地址。
3. 浏览器刷新且 token 存在：进入布局后请求 `/auth/info` 恢复用户名。
4. token 过期、签名无效或账号失效：后端返回 401；响应拦截器清除本地会话并跳转登录。
5. 用户主动退出：立即清除本地会话并进入登录页，不等待服务端请求。

## 测试与验收

- 前端纯函数测试：未登录访问根布局子页面会跳登录；已登录访问视频页面不跳登录；登录回跳只接受安全站内路径。
- 后端单元测试：令牌 `exp - iat` 等于配置有效期；登录响应中的 `expires_in` 使用同一配置值。
- 前端完整测试与生产构建通过。
- 后端认证相关测试通过。
- 重启后端后验证 8090 服务健康；登录响应返回 `expires_in: 604800`，登录后可在各业务页面间切换，退出后进入登录页。

## 范围边界

- 本次不改变现有用户角色和管理员权限模型。
- 本次不新增刷新令牌；开发阶段采用单个 7 天访问令牌。
- 本次不把所有传感器后端接口改为强制认证，只修正系统入口与会话体验，并保留视频接口现有认证。

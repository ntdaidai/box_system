# 风速历史图横轴与悬浮点设计

## 目标

让风速历史月视图与温度历史图保持一致的日期表达，并避免低风速数据在折线上常驻显示圆点。

## 范围

- 风速月视图横轴只显示日期，不显示风向。
- 月视图固定显示 `1日、5日、9日、13日、17日、21日、25日、29日`。
- 风速年度视图继续按月份显示，不改变近 24 小时视图。
- 折线默认不显示数据圆点；鼠标悬浮到数据点时仍显示高亮圆点和现有提示框。
- 提示框继续显示每日平均风速、风级和风向。

## 实现设计

在 `windHistory.js` 中增加两个纯函数：一个负责格式化月/年横轴标签，另一个负责判断日历轴上的标签是否需要展示。月视图按日号的四天步长判断，年度视图复用现有的月首与窄屏隔月规则。

`SensorWind.vue` 使用这两个函数生成轴标签和间隔，并把风速折线的 `showSymbol` 固定为 `false`。保留 `symbol`、`symbolSize`、`itemStyle` 和 `emphasis`，由 ECharts 在悬浮时绘制高亮圆点。

## 测试与验收

- 单元测试验证月视图仅生成带“日”的日期标签，且只选择 1、5、9……29 日。
- 单元测试验证年度视图继续按月份筛选标签。
- 源码契约测试验证折线配置为 `showSymbol: false`，防止低风速分支重新显示常驻圆点。
- 运行前端全部 Node 测试与 Vite 生产构建。

## 非目标

- 不修改风速接口、数据聚合或提示框内容。
- 不重构其他传感器图表，也不改变颜色、曲线或纵轴样式。
